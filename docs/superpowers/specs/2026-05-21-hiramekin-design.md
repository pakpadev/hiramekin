**作成日**: 2026-05-21  
**バージョン**: 1.0  
**ステータス**: 承認済み

---

## 概要

閃筋（ひらめきん）は、一瞬のひらめきやアイディアを瞬時に書き留めることができる殴り書きアプリ。簡単な計算も同一画面で行える。アイデアの記録と簡単な計算を「最短最速」で行うことを第一原則とする。

---

## プロダクトビジョン

**第一原則**: アプリを開いたら即座に入力できる状態にある

**v2 最重要機能**: Androidホーム画面ウィジェット（0タップで入力）  
ウィジェット対応はv1設計段階から構造を考慮する。後付けが困難なため、bare workflowを採用する理由の一つ。

| 項目 | 採用技術 | 理由 |
| --- | --- | --- |
| フレームワーク | Expo bare workflow (React Native) | ウィジェット(v2)対応のためbareを選択 |

| ストレージ (モバイル) | expo-sqlite または AsyncStorage | オフライン完結 |

| ナビゲーション | React Navigation (Stack) | シンプルな前後移動のみ |

## 画面構成

### HomeScreen（メモ一覧）

```text
+---------------------------+
| [検索バー]                |  上部固定
+---------------------------+
| [ピン留め]                |  ピン留めメモのみ表示
|  - メモA        15:32     |
|  - メモB        昨日      |
+---------------------------+
| [最新順]                  |  時系列リスト
|  - メモC        14:00     |
|  - メモD        5/20      |
+---------------------------+
                    [+]       FABボタン、常時表示
```

- アプリ起動時はこの画面から直接表示（スプラッシュなし）
- FABタップ → MemoScreen（新規）
- メモタップ → MemoScreen（編集）

### MemoScreen（メモ入力・編集）

```text
+---------------------------+
| [ピン留めトグル]  [...]    |  ヘッダー
+---------------------------+
|                           |
| ふつうのテキスト           |
| 100 + 200 = [300]         |  <- 数式行：結果をリアルタイム表示
| 売上 1000 * 0.08 = [80]   |  <- タップで数式/結果のみ切替
| 1000                      |  <- 数字だけの行は計算対象外
|                           |
+---------------------------+
```

- 起動時カーソル自動フォーカス、キーボード自動展開
- 保存ボタンなし（テキスト変更のたびに自動保存）
- 数式行の `[結果]` 部分をタップ → 数式ごと結果のみ表示に切替（再タップで元に戻る）

### SearchScreen（検索）

- 検索バータップで遷移
- 入力に応じてリアルタイムフィルタリング（content の部分一致）

### SettingsScreen（設定）

- テーマ切り替え（v1はOS設定に従う固定で設定項目なし、将来用に画面のみ用意）

## 計算エンジン仕様

### 対応演算

| 記法 | 例 | 結果 |
| --- | --- | --- |
| 四則演算 | `100 + 200` | `300` |
| パーセント | `1000 * 0.08` | `80` |
| 小数 | `3.14 * 2` | `6.28` |
| 括弧 | `(100 + 200) * 3` | `900` |

### 非対応（v1）

- 行をまたいだ変数参照
- 関数（sin, sqrt など）

### 数式行の判定

- 行に数字と演算子（`+`, `-`, `*`, `/`, `(`, `)`）が含まれる場合のみ評価
- 評価失敗・不正数式の場合は結果欄を空白にする（エラーメッセージ表示なし）
- 結果はテキストに書き込まず、行の右隣にオーバーレイ表示する（ユーザーの入力テキストを変更しない）

---

## エラーハンドリング方針

| ケース | 対応 |
| --- | --- |
| 不正な数式 | 結果欄を空にして無視、クラッシュしない |
| ストレージ書き込み失敗 | メモリキャッシュで保持、次回起動時に再試行 |
| アプリ強制終了 | リアルタイム自動保存のためデータ損失なし |
| PWAオフライン | ローカルのみのためオフラインで全機能利用可 |

## UIデザイン方針

- テーマ: OS設定に従うダーク/ライト自動切替
- フォント: システムフォント（追加フォントなし、軽量化）
- アニメーション: 最小限（画面遷移のみ）
- ナビゲーションバー: なし（Stack のヘッダーのみ）
- カラー: Neutral グレーベース、アクセントカラー1色

## 非機能要件

- アプリ起動 → 入力可能状態まで 1秒以内
- メモ自動保存レイテンシ: 500ms 以内（デバウンス）
- 検索レスポンス: 200ms 以内（ローカル検索のため）
- 最小サポートバージョン: Android 10 / iOS 15 / 主要ブラウザ最新2世代
