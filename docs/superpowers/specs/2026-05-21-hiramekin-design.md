**作成日**: 2026-05-21

---

## 概要

閃筋（ひらめきん）は、一瞬のひらめきやアイディアを瞬時に書き留めることができる殴り書きアプリ。簡単な計算も同一画面で行える。アイデアの記録と簡単な計算を「最短最速」で行うことを第一原則とする。

---

## プロダクトビジョン

**第一原則**: アプリを開いたら即座に入力できる状態にある

**v2 最重要機能**: Androidホーム画面ウィジェット（0タップで入力）  
ウィジェット対応はv1設計段階から構造を考慮する。後付けが困難なため、bare workflowを採用する理由の一つ。

## 画面構成（単一画面・状態ベース）

画面遷移なし。`editingMemoId` と `searchQuery` の状態によってUIが切り替わる。

### 状態: browsing（初期状態・起動時）

```text
+---------------------------+
| 新規メモ入力エリア         |  起動時カーソル自動フォーカス
| (1行分、入力で拡張)        |
+---------------------------+
| [検索バー]                |
+---------------------------+
| [ピン留め]                |
|  - メモA        15:32     |
|  - メモB        昨日      |
+---------------------------+
| [最新順]                  |
|  - メモC        14:00     |
|  - メモD        5/20      |
+---------------------------+
                    [+]       新規メモ開始（入力エリアにフォーカス）
```

### 状態: editing（メモをタップ時）

```text
+---------------------------+
| メモ内容が入力エリアに展開 |  editingMemoId がセットされる
| ふつうのテキスト           |
| 100 + 200 = [300]         |  リアルタイム計算
| 売上 1000 * 0.08 = [80]   |  [結果]タップで切替
|                           |
| [ピン留めトグル]  [削除]   |
+---------------------------+
  キーボードを閉じる or 入力エリア外タップ → browsing に戻る
  （離脱時に自動保存）
```

### 状態: searching（検索バーに入力時）

```text
+---------------------------+
| 新規メモ入力エリア         |
+---------------------------+
| [検索バー: "キーワード"]   |  入力中
+---------------------------+
| 検索結果リスト             |  リアルタイムフィルタリング
|  - マッチしたメモA         |
|  - マッチしたメモB         |
+---------------------------+
  検索バークリア → browsing に戻る
```

## 状態遷移

```text
起動
 └─ browsing
      ├─ 入力エリアにタイプ → 新規メモ作成（editingMemoId = null のまま）
      ├─ メモをタップ → editing（editingMemoId = タップしたメモのid）
      ├─ 検索バーに入力 → searching
      └─ FABタップ → 入力エリアにフォーカス（browsing のまま）

editing
 └─ キーボードを閉じる / 入力エリア外タップ
      └─ 自動保存 → browsing

searching
 └─ 検索バークリア / キャンセル → browsing
 └─ 検索結果のメモをタップ → editing
```

## 計算エンジン仕様

### 対応演算

| 記法 | 例 | 結果 |
| --- | --- | --- |
| 四則演算 | `100 + 200` | `300` |
| パーセント | `1000 * 0.08` | `80` |
| 小数 | `3.14 * 2` | `6.28` |
| 括弧 | `(100 + 200) * 3` | `900` |

### 非対応（v1）

- 行をまたいだ変数参照
- 関数（sin, sqrt など）

### 数式行の判定

- 行に数字と演算子（`+`, `-`, `*`, `/`, `(`, `)`）が含まれる場合のみ評価
- 評価失敗・不正数式の場合は結果欄を空白にする（エラーメッセージ表示なし）
- 結果はテキストに書き込まず、行の右隣にオーバーレイ表示する（ユーザーの入力テキストを変更しない）
- 数式行の `[結果]` 部分をタップ → 数式ごと結果のみ表示に切替（再タップで元に戻る）

## 自動保存ロジック

```text
新規メモ:
  入力開始と同時にIDを発行してストレージに保存
  以降は500msデバウンスで差分保存

既存メモ編集:
  テキスト変更のたびに500msデバウンスで保存
  editingモード離脱時（キーボードclose / 外タップ）に即時保存

空のメモ:
  入力エリアが空のままeditingモードを離脱した場合は保存しない
  既存メモを空にした場合は削除確認なしで削除
```

---

## エラーハンドリング方針

| ケース | 対応 |
| --- | --- |
| 不正な数式 | 結果欄を空にして無視、クラッシュしない |
| ストレージ書き込み失敗 | メモリキャッシュで保持、次回起動時に再試行 |
| アプリ強制終了 | リアルタイム自動保存のためデータ損失なし |
| PWAオフライン | ローカルのみのためオフラインで全機能利用可 |

## UIデザイン方針

- テーマ: OS設定に従うダーク/ライト自動切替

## SettingsScreen（設定）

単一画面構成の例外として設定画面のみ別画面として保持する。  
アクセス頻度が極めて低いため、画面遷移コストが問題にならない。  
v1は設定項目なし（将来用に画面だけ用意）。

## 非機能要件

- アプリ起動 → 入力可能状態まで 1秒以内
- メモ自動保存レイテンシ: 500ms 以内（デバウンス）
- 検索レスポンス: 200ms 以内（ローカル検索のため）
- 最小サポートバージョン: Android 10 / iOS 15 / 主要ブラウザ最新2世代
