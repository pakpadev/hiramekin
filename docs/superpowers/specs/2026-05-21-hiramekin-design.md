# 閃筋 (Hiramekin) 設計仕様書

**作成日**: 2026-05-21  
**バージョン**: 1.3  
**ステータス**: 承認済み

---

## 概要

閃筋（ひらめきん）は、一瞬のひらめきやアイディアを瞬時に書き留めることができる殴り書きアプリ。簡単な計算も同一画面で行える。アイデアの記録と簡単な計算を「最短最速」で行うことを第一原則とする。

---

## プロダクトビジョン

**第一原則**: アプリを開いたら即座に入力できる状態にある

**v2 最重要機能**: Androidホーム画面ウィジェット（0タップで入力）  
ウィジェット対応はv1設計段階から構造を考慮する。後付けが困難なため、bare workflowを採用する理由の一つ。

## データモデル

```typescript
interface Memo {
  id: string         // UUID
  content: string    // テキスト本文（数式込み）
  isPinned: boolean  // ピン留め
  isArchived: boolean // アーカイブ（通常リストから非表示）
  createdAt: number  // Unix timestamp (ms)
  updatedAt: number  // Unix timestamp (ms)
}

interface AppSettings {
  theme: 'system' | 'light' | 'dark' // v1は 'system' 固定
}

// 画面状態（永続化しない）
interface UIState {
  editingMemoId: string | null // null = 新規作成モード
  searchQuery: string
  showArchive: boolean         // アーカイブ一覧表示中
  isListening: boolean         // 音声入力中
}
```

### Supabase スキーマ（v1では使用しない、定義のみ）

```sql
create table memos (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users,
  content text not null,
  is_pinned boolean default false,
  is_archived boolean default false,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

## 画面構成（単一画面・状態ベース）

画面遷移なし。`editingMemoId` と `searchQuery` の状態によってUIが切り替わる。

### 状態: browsing（初期状態・起動時）

```text
+---------------------------+
| 新規メモ入力エリア         |  起動時カーソル自動フォーカス
| (1行分、入力で拡張)        |
+---------------------------+
| [検索バー]                |
+---------------------------+
| [ピン留め]                |
|  - メモA        15:32     |
|  - メモB        昨日      |
+---------------------------+
| [最新順]                  |
|  - メモC        14:00     |
|  - メモD        5/20      |
+---------------------------+
                    [+]       新規メモ開始（入力エリアにフォーカス）
```

### 状態: editing（メモをタップ時）

```text
+---------------------------+
| メモ内容が入力エリアに展開 |  editingMemoId がセットされる
| ふつうのテキスト           |
| 100 + 200 = [300]         |  リアルタイム計算
| 売上 1000 * 0.08 = [80]   |  [結果]タップで切替
|                           |
| [ピン留めトグル]  [アーカイブ] |
+---------------------------+
| [今日] [時刻] [マイク]    |  キーボード上部ツールバー
+--[キーボード]--------------+
  キーボードを閉じる or 入力エリア外タップ → browsing に戻る
  （離脱時に自動保存）
```

### 状態: searching（検索バーに入力時）

```text
+---------------------------+
| 新規メモ入力エリア         |
+---------------------------+
| [検索バー: "キーワード"]   |  入力中
+---------------------------+
| 検索結果リスト             |  リアルタイムフィルタリング
|  - マッチしたメモA         |
|  - マッチしたメモB         |
+---------------------------+
  検索バークリア → browsing に戻る
```

## 状態遷移

```text
起動
 └─ browsing
      ├─ 入力エリアにタイプ → 新規メモ作成（editingMemoId = null のまま）
      ├─ メモをタップ → editing（editingMemoId = タップしたメモのid）
      ├─ 検索バーに入力 → searching
      └─ FABタップ → 入力エリアにフォーカス（browsing のまま）

editing
 └─ キーボードを閉じる / 入力エリア外タップ
      └─ 自動保存 → browsing

searching
 └─ 検索バークリア / キャンセル → browsing
 └─ 検索結果のメモをタップ → editing
```

## キーボードツールバー（クイック挿入）

キーボード展開時に上部に表示されるツールバー。カーソル位置にテキストを挿入する。

| ボタン | 動作 | 用途 |
| --- | --- | --- |
| 今日 | `2026/05/21` をカーソル位置に挿入 | 日付メモの起点 |
| 時刻 | `22:30` をカーソル位置に挿入 | 時刻メモの起点 |
| マイク | 音声入力を開始 | ハンズフリー入力 |

### 日付・時刻の編集方針

過去・未来の日付を入力する場合はカレンダーUIを使わない。「今日」ボタンで今日の日付を挿入し、変えたい数字だけ手動で修正する。

```text
例: 昨日の日付を入力したい場合
  「今日」タップ → 2026/05/21 が挿入される
  「21」を「20」に手動修正 → 2026/05/20
```

日付ピッカーはコンセプト（最短最速）に反するため採用しない。

## 計算エンジン仕様

### 対応演算

| 記法 | 例 | 結果 |
| --- | --- | --- |
| 四則演算 | `100 + 200` | `300` |
| パーセント | `1000 * 0.08` | `80` |
| 小数 | `3.14 * 2` | `6.28` |
| 括弧 | `(100 + 200) * 3` | `900` |

### 非対応（v1）

- 行をまたいだ変数参照
- 関数（sin, sqrt など）

### 数式行の判定

- 行に数字と演算子（`+`, `-`, `*`, `/`, `(`, `)`）が含まれる場合のみ評価
- 評価失敗・不正数式の場合は結果欄を空白にする（エラーメッセージ表示なし）
- 結果はテキストに書き込まず、行の右隣にオーバーレイ表示する（ユーザーの入力テキストを変更しない）
- 数式行の `[結果]` 部分をタップ → 数式ごと結果のみ表示に切替（再タップで元に戻る）

## 音声入力

キーボードツールバーのマイクボタンで起動。OSネイティブの音声認識（Android: SpeechRecognizer、iOS: SFSpeechRecognizer）を `expo-speech` 経由で呼び出す。

```text
マイクタップ
 └─ 音声入力オーバーレイ表示（isListening = true）
      ├─ 認識中: 波形アニメーション表示
      ├─ 認識テキスト確定 → カーソル位置に挿入、オーバーレイ非表示
      └─ キャンセル / タイムアウト → オーバーレイ非表示
```

- 認識結果はテキストとしてそのまま挿入（変換確認なし）
- 音声入力後も通常のテキスト編集が可能
- PWA（デスクトップ）では Web Speech API にフォールバック

## アーカイブ

削除の代わりにアーカイブへ退避する。通常リストには表示されないが失われない。

### 操作

- editingモードのヘッダー「アーカイブ」ボタンタップ → 即アーカイブ（確認ダイアログなし）
- アーカイブ一覧は SettingsScreen 内の「アーカイブを見る」からアクセス

### 自動削除ルールの変更

| ケース | 旧動作 | 新動作 |
| --- | --- | --- |
| 既存メモを空にして離脱 | 即削除 | アーカイブに移動 |
| 新規メモを空のまま離脱 | 保存しない | 保存しない（変わらず） |

### アーカイブ一覧

```text
SettingsScreen
 └─ アーカイブを見る
      ├─ アーカイブ済みメモの一覧（時系列）
      ├─ タップ → 復元（isArchived = false）
      └─ 完全削除ボタン（長押しで確認）
```

## ハプティクスフィードバック

`expo-haptics` を使用。モバイルのみ有効（PWAでは無視）。

| タイミング | 種類 | 意味 |
| --- | --- | --- |
| 自動保存完了 | Light Impact | 「捕まえた」の確認 |
| アーカイブ実行 | Medium Impact | 操作の区切り |
| ピン留めトグル | Selection Changed | 状態変更 |

UIアニメーションを使わずに操作の手応えを伝えるため、アニメーション「最小限」方針と矛盾しない。

---

## 自動保存ロジック

```text
新規メモ:
  入力開始と同時にIDを発行してストレージに保存
  以降は500msデバウンスで差分保存

既存メモ編集:
  テキスト変更のたびに500msデバウンスで保存
  editingモード離脱時（キーボードclose / 外タップ）に即時保存

空のメモ:
  入力エリアが空のままeditingモードを離脱した場合は保存しない（新規のみ）
  既存メモを空にして離脱した場合はアーカイブに移動（即削除しない）
```

---

## エラーハンドリング方針

| ケース | 対応 |
| --- | --- |
| 不正な数式 | 結果欄を空にして無視、クラッシュしない |
| ストレージ書き込み失敗 | メモリキャッシュで保持、次回起動時に再試行 |
| アプリ強制終了 | リアルタイム自動保存のためデータ損失なし |
| PWAオフライン | ローカルのみのためオフラインで全機能利用可 |

## UIデザイン方針

- テーマ: OS設定に従うダーク/ライト自動切替
- フォント: システムフォント（追加フォントなし、軽量化）

- カラー: Neutral グレーベース、アクセントカラー1色

## SettingsScreen（設定）

単一画面構成の例外として設定画面のみ別画面として保持する。  
アクセス頻度が極めて低いため、画面遷移コストが問題にならない。  
v1は設定項目なし（将来用に画面だけ用意）。

## v2 ロードマップ（本仕様の対象外）

| 優先度 | 機能 |
| --- | --- |
| 最重要 | Androidホーム画面ウィジェット |
| 高 | Supabaseによるデバイス間同期 |
| 中 | タグ機能 |
| 中 | iOSウィジェット（Swiftが必要） |
| 低 | 行をまたいだ変数参照 |

## 非機能要件

- アプリ起動 → 入力可能状態まで 1秒以内
- メモ自動保存レイテンシ: 500ms 以内（デバウンス）
- 検索レスポンス: 200ms 以内（ローカル検索のため）
- 最小サポートバージョン: Android 10 / iOS 15 / 主要ブラウザ最新2世代
